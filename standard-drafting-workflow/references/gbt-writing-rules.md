# GB/T 1.1—2020 起草规则细则

本文件根据用户提供的 GB/T 1.1—2020 MinerU/OCR Markdown 整理，供标准编写流程读取。内容为规则化转述，不作为标准原文替代；涉及技术正确性、合规边界、计量检测、安全健康等内容仍需专家确认。

## 使用边界

- 本文件记录 GB/T 1.1—2020 中与标准结构、起草、表述、引用、图表公式、附录、专利商标、安全提示和编排格式有关的规则。
- md2std 语法、DOCX 模板字段、Word XML、COM 后处理等生成器实现细节不写入本文件。
- 审校时，机械格式问题先由 `md2std-standard-auditor` 处理；本文件用于人工/AI 判断机械规则覆盖不到的起草质量问题。

## 1. 目标、原则和标准功能类型

### 1.1 编制目标

- 文件应通过清楚、准确、无歧义的条款，为未来技术发展提供框架，并能被未参加编制的专业人员理解和应用。
- 起草前应分析标准化需求、最新技术水平、市场情况、标准化对象、文件使用者和编制目的。
- 起草时应先明确文件类别/功能类型，再选择规范性要素、层次结构和技术内容。

### 1.2 整体文件或分部分文件

- 一个标准化对象通常宜编制为一个整体文件。
- 可分为若干部分的典型情形：篇幅过长；使用者明显不同；编制目的不同。
- 适用范围广的通用内容宜形成整体文件；较窄对象的通用内容可形成通用部分；单一对象的具体特性通常作为相关要素，而不宜单独编为整体文件或某个部分。
- 起草分部分文件前，应确定分部分原因、各部分关系、每个部分的名称和范围。
- 分部分文件的各部分共用同一文件顺序号；每个部分可单独编制、修订和发布；部分不应再细分为分部分。
- 分部分可按对象的特殊方面划分，也可按通用方面和特殊方面划分。特殊方面可修改或补充通用方面，但不宜脱离通用部分单独使用。

### 1.3 规范性要素选择原则

- 标准化对象原则：确认拟标准化的是产品/系统、过程、服务，还是某领域相关内容；确认是完整对象还是对象的某个方面。
- 文件使用者原则：确认文件服务的使用者类型，以及使用者关注结果还是过程。
- 目的导向原则：基于编制目的进行功能分析，识别需要标准化的内容或特性。
- 编制目的越多，通常需要选取的内容或特性越多；仍应避免堆砌与目的无关的内容。

### 1.4 表述原则

- 一致性：相同条款宜使用相同用语；类似条款宜使用类似用语；同一概念宜使用同一术语；相似要素的标题和编号宜尽可能一致。
- 协调性：与现行有效文件保持协调，避免重复和不必要差异；通用内容宜集中在通用文件或通用部分中；需要使用其他位置或其他文件内容时，宜采用引用或提示。
- 易用性：内容应便于直接应用，便于其他文件引用或剪裁使用。

### 1.5 总体要求

- 起草文件时应在选择规范性要素的基础上确定预计结构和内在关系。
- 不同功能类型标准应符合 GB/T 20001 相应部分；特定内容应符合 GB/T 20002 相应部分；采用国际文件时应符合 GB/T 20000.2。
- 文件中不应规定索赔、担保、费用结算等合同要求。
- 文件中不应规定行政管理措施、法律责任、罚则等法律法规要求。

### 1.6 功能类型和核心内容

| 功能类型 | 典型核心技术要素 | 条款类型重点 |
| --- | --- | --- |
| 术语标准 | 术语条目 | 定义使用陈述型条款 |
| 符号标准 | 符号/标志及其含义 | 符号含义使用陈述型条款 |
| 分类标准 | 分类和/或编码 | 陈述、要求型条款 |
| 试验标准 | 试验步骤、试验数据处理 | 步骤使用指示/要求型条款，数据处理使用陈述/指示型条款 |
| 规范标准 | 要求、证实方法 | 要求型条款，证实方法使用指示/陈述型条款 |
| 规程标准 | 程序、追溯/证实方法 | 程序确立用陈述型条款，程序执行用指示/要求型条款 |
| 指南标准 | 需考虑因素 | 推荐、陈述型条款 |

## 2. 文件名称与结构

### 2.1 文件名称

- 文件名称应清晰、简明地描述文件覆盖的主题，使文件易于与其他文件区分。
- 文件名称不应包含不必要细节；必要补充说明应由范围给出。
- 文件名称通常由引导元素、主体元素、补充元素组成。
- 引导元素用于指出文件所属领域；当主体元素已经能明确领域时，可省略。
- 主体元素应给出标准化对象，不应缺失。
- 补充元素用于指出文件所覆盖的标准化对象的特殊方面，或文件与其他文件/部分的区别。
- 文件名称不宜用过窄措辞限制范围。范围边界应在 `范围` 中说明。
- 文件名称中的词语应与范围、核心技术要素和标准化对象保持一致。

### 2.2 层次

文件层次通常包括：

| 层次 | 编号示例 | 说明 |
| --- | --- | --- |
| 部分 | `XXXX.1` | 文件分为多个部分时使用 |
| 章 | `5` | 基本层次 |
| 条 | `5.1`、`5.1.1` | 章内细分，可继续细分 |
| 段 | 无编号 | 章或条内无编号细分 |
| 列项 | `—`、`·`、`a)`、`1)` | 段中的并列细分 |

### 2.3 要素类别和表述形式

- 按作用可分为规范性要素和资料性要素。
- 按存在状态可分为必备要素和可选要素。
- 规范性要素主要由条款构成，可包含少量附加信息。
- 资料性要素由附加信息构成。
- 要素内容通常用条文表述；必要时可使用附录、图、表、数学公式、注、脚注、引用、提示等形式。

| 要素 | 必备/可选 | 规范性/资料性 | 主要构成和表述 |
| --- | --- | --- | --- |
| 封面 | 必备 | 资料性 | 标明文件信息 |
| 目次 | 可选 | 资料性 | 自动生成的列表 |
| 前言 | 必备 | 资料性 | 条文、注、脚注、指明附录 |
| 引言 | 可选 | 资料性 | 条文、图、表、公式、注、脚注、指明附录 |
| 范围 | 必备 | 规范性 | 条款、附加信息；可用条文、表、注、脚注 |
| 规范性引用文件 | 必备/可选 | 资料性 | 引导语和文件清单 |
| 术语和定义 | 必备/可选 | 规范性 | 条款、附加信息；可用条文、图、公式、示例、注、引用、提示 |
| 符号和缩略语 | 可选 | 规范性 | 条款、附加信息 |
| 分类和编码/系统构成 | 可选 | 规范性 | 条款、附加信息 |
| 总体原则和/或总体要求 | 可选 | 规范性 | 条款、附加信息 |
| 核心技术要素 | 必备 | 规范性 | 条款、附加信息 |
| 其他技术要素 | 可选 | 规范性 | 条款、附加信息 |
| 参考文献 | 可选 | 资料性 | 文件清单、脚注 |
| 索引 | 可选 | 资料性 | 自动生成的列表 |

### 2.4 要素选择和顺序

- 规范性要素中，范围、术语和定义、核心技术要素是必备要素；术语和定义的内容可根据具体情况为空，但应给出相应说明。
- 规范性引用文件为资料性要素，但在结构中属于标准正文前部的必备要素；其内容有无根据文件引用情况确定。
- 资料性要素的位置、先后顺序和标题应与 GB/T 1.1 的结构保持一致。
- 正文技术要素之后，如有附录，应先设置附录；参考文献置于最后一个附录之后；索引作为最后一个要素。
- 附录不应放在参考文献之后。

推荐顺序：

```text
封面 -> 目次 -> 前言 -> 引言 -> 范围 -> 规范性引用文件
-> 术语和定义 -> 符号和缩略语 -> 分类和编码/系统构成
-> 总体原则和/或总体要求 -> 核心技术要素 -> 其他技术要素
-> 附录 -> 参考文献 -> 索引 -> 封底
```

## 3. 层次编写规则

### 3.1 章

- 章是文件层次划分的基本单元。
- 章编号应从 `范围` 一章开始，使用从 1 开始的阿拉伯数字，连续编号到附录之前。
- 每一章均应有章标题，并置于编号之后。

### 3.2 条

- 条是章内有编号的细分层次，可以继续细分，但层次不宜过多，最多分到第五层次。
- 一个层次中有一个以上的条时才设置条；没有并列条时不应只设置一个条。
- 条编号使用阿拉伯数字，用下脚点与上一层次编号相隔。
- 第一层次的条宜给出条标题；第二层次可同样处理。同一章或条的下一层次中，各条有无标题应一致。
- 无标题条的首句可用黑体突出关键术语或短语；同一层次的无标题条处理方式应一致。
- 无标题条不应再分条。

### 3.3 段

- 段是章或条内无编号的细分层次。
- 为避免引用混淆，不宜在章标题与条之间或条标题与下一层次条之间设置悬置段。
- 术语和定义、符号和缩略语中的引导语，以及重要提示，不属于悬置段。
- 如悬置段可能导致引用混淆，可改为有编号条、移动位置或删除。

### 3.4 列项

- 列项用于段中的并列细分，由引语和并列项组成。
- 引语为完整句并以句号结束时，各列项可用句号结束。
- 引语以后跟冒号的文字引出时，列项可用分号或逗号连接；最后一项以句号结束。
- 列项可进一步细分为分项，但不宜超过两个层次。
- 列项符号可用破折号或间隔号；列项编号可用后带半圆括号的小写拉丁字母或阿拉伯数字，如 `a)`、`1)`。
- 通常第一层次使用破折号，第二层次使用间隔号；需要识别顺序时可用字母编号，进一步细分时可用间隔号或数字编号。
- 可用黑体突出列项中的关键术语或短语。

## 4. 各要素编写规则

### 4.1 封面

- 封面用于标明文件信息。
- 必备信息包括文件名称、文件层次或类别、文件代号、文件编号、ICS 号、CCS 号、发布日期、实施日期、发布机构等。
- 如文件代替一个或多个文件，应在封面标明被代替文件编号；被代替文件较多时不宜超过一行，不能一行列出时可在前言说明。
- 与国际文件有一致性对应关系时，应在封面标示一致性程度。
- 国家标准、行业标准封面还应标明文件名称英文译名；行业标准封面还应标明备案号。
- 征求意见稿和送审稿封面显著位置应给出征集专利信息的提示。

### 4.2 目次

- 目次用于呈现文件结构，通常有必要设置。
- 目次宜自动生成。
- 目次可依次列出前言、引言、章编号和标题、需要时的条编号和标题、附录编号和属性及标题、需要时的附录条、参考文献、索引、需要时的图编号和图题、需要时的表编号和表题。
- 目次应给出所在页码。
- 目次中不应列出 `术语和定义` 中的条目编号和术语。

### 4.3 前言

- 前言给出文件自身内容之外的信息，如起草依据、文件关系、编制和起草者信息。
- 前言不应包含要求、指示、推荐或允许型条款。
- 前言不应使用图、表或数学公式。
- 前言不应给出章编号且不分条。
- 前言中宜依次给出：起草依据；与其他文件的关系；与代替文件的关系；与国际文件关系；专利说明；提出和归口信息；起草单位和主要起草人；历次版本发布情况。
- 起草依据通常说明本文件按照 GB/T 1.1—2020 的规定起草。
- 首次发布的文件，历次版本发布情况宜统一表述为 `本文件为首次发布。`
- 代替、废止或修订文件时，应说明本文件及其所代替或废止文件的历次版本发布情况；不应把首次发布、修订和代替关系混写成含糊表述。

### 4.4 引言

- 引言说明与文件自身内容相关的背景信息，不应包含要求型条款。
- 分为部分的文件，或文件某些内容涉及专利时，通常设置引言。
- 引言不应给出章编号；如需分条，仅对条编号，编号为 `0.1`、`0.2` 等。
- 引言通常说明编制原因、编制目的、分为部分的原因、各部分关系、技术内容特殊信息等。
- 已识别出文件内容涉及专利时，应在引言中说明；内容较多时可移作附录。
- 不分部分、未涉及专利且无必要背景信息时，不宜仅为补充泛泛背景而设置引言；确需设置时，应能说明与文件自身内容直接相关的编制原因、目的或特殊技术背景。

### 4.5 范围

- 范围界定文件的标准化对象、覆盖方面和适用界限；必要时指出通常可能覆盖但实际不涉及的内容。
- 分为部分的文件，各部分范围只界定各自部分的标准化对象和覆盖方面。
- 范围应设置为第 1 章；确有必要时可进一步细分为条。
- 范围陈述应简洁，可作为内容提要使用。
- 范围不应陈述可在引言中给出的背景信息。
- 范围应使用陈述型条款，不应包含要求、指示、推荐或允许型条款。
- 范围可使用“本文件规定了……”“本文件确立了……”“本文件描述了……”“本文件提供了……”“本文件给出了……”“本文件界定了……”等表述。
- 适用界限可用“本文件适用于……”“本文件不适用于……”等表述。

### 4.6 规范性引用文件

- 规范性引用文件要素列出文件中规范性引用的文件，由引导语和文件清单构成。
- 该要素应设置为第 2 章，且不应分条。
- 有规范性引用文件时，应使用固定导语，说明注日期引用仅对应版本适用，不注日期引用适用最新版本及修改单。
- 没有规范性引用文件时，应写明：`本文件没有规范性引用文件。`
- 文件清单应列出规范性引用的每个文件，列出的文件之前不给序号。
- 注日期引用给出文件代号、顺序号、发布年份号和/或月份号以及文件名称。
- 注日期引用中，顺序号与年份号之间应使用一字线连接号 `—`，不应使用半字线 `-` 或短横线类字符；例如写作 `GB/T 1.1—2020`，不写作 `GB/T 1.1-2020`。
- 不注日期引用给出文件代号、顺序号以及文件名称。
- 不注日期引用所有部分时，应给出文件代号、顺序号、`（所有部分）` 和文件名称中必要的引导元素、主体元素。
- 国际文件或国外出版物应给出编号或代号、顺序号、中文译名，并在括号中给出原文名称。
- 标准化文件之外的引用文件和信息资源应遵守 GB/T 7714。
- 引用文件排序：
  1. 国家标准化文件
  2. 行业标准化文件
  3. 本行政区域地方标准化文件（仅地方标准适用）
  4. 团体标准化文件
  5. ISO、ISO/IEC 或 IEC 标准化文件
  6. 其他机构或组织的标准化文件
  7. 其他文献
- 国家标准、ISO 或 IEC 标准按文件顺序号排列；行业、地方、团体和其他国际标准化文件先按文件代号排序，再按顺序号排序。
- 排序示例：`GB 5749`、`GB/T 5750.6`、`GB/T 5750.12`、`GB/T 11615`、`GB/T 14848` 应按顺序号从小到大排列；`DZ/T 0481`、`DZ/T 0482` 等行业标准应列在国家标准之后，并按文件代号和顺序号排列。

### 4.7 术语和定义

- 术语和定义要素界定理解文件所必需的术语，由引导语和术语条目构成。
- 该要素应设置为第 3 章；为了表示概念分类可以细分为条，每条应给出标题。
- 仅本要素界定术语时，可用“下列术语和定义适用于本文件。”
- 仅引用其他文件术语时，可用“……界定的术语和定义适用于本文件。”
- 同时引用其他文件并列出本文件术语时，可用“……界定的以及下列术语和定义适用于本文件。”
- 没有需要界定的术语和定义时，应写明：`本文件没有需要界定的术语和定义。`
- 术语条目宜按概念层级分类和编排；不能或无须分类时可按汉语拼音字母顺序编排。
- 术语条目编号是条目编号，不是条编号。
- 每个术语条目通常包括条目编号、术语、英文对应词、定义；需要时可增加符号、概念其他表述形式、示例、注、来源等。
- 符号来自国际权威组织时，宜在同一行方括号中标出组织名称或缩略语。
- 图和数学公式可作为定义辅助形式。
- 术语条目不应编排成表，不准许插入脚注。
- 需要定义的术语应同时满足：文件中至少使用两次；专业使用者在不同语境中理解不一致；尚无定义或需要改写已有定义；属于文件范围限定领域。
- 范围外术语可在条文注中说明含义，不宜在术语和定义中界定。
- 宜界定一般概念术语，不宜界定可由一般概念组合而成的具体概念术语。
- 定义宜能在上下文中替代术语，优选内涵定义：“区别特征 + 上位概念”。
- 定义应使用陈述型条款，不应包含要求型条款，也不应写成要求形式。
- 定义中的附加信息应以示例或注表述。
- 抄录其他文件少量术语条目时，应准确标明来源；改写定义时应说明“有修改”。

### 4.8 符号和缩略语

- 符号和缩略语要素给出理解文件所必需的符号、缩略语说明或定义。
- 需要设置时，宜作为第 4 章。
- 如符号需反映技术准则的特定次序，该要素可细分为条，每条给出标题。
- 可根据需要并入 `术语和定义`。
- 引导语按内容选择：“下列符号适用于本文件。”“下列缩略语适用于本文件。”“下列符号和缩略语适用于本文件。”
- 清单中的符号和缩略语之前不给序号。
- 符号和缩略语宜按字母顺序列出：大写拉丁字母在小写之前；无角标字母在有角标字母之前；有字母角标的在有数字角标的之前；希腊字母在拉丁字母之后；其他特殊符号置于最后。
- 说明或定义宜使用陈述型条款，不应包含要求和推荐型条款。

### 4.9 分类和编码/系统构成

- 分类和编码用于给出标准化对象的划分，以及分类结果的命名或编码。
- 系统标准通常设置系统构成，用于确立构成系统的分系统或组成单元。
- 分类和编码/系统构成通常使用陈述型条款。
- 可根据编写需要与规范、规程或指南标准的核心技术要素合并。

### 4.10 总体原则和/或总体要求

- 总体原则用于规定为达到编制目的需要依据的方向性总框架或准则。
- 总体要求用于规定涉及整体文件或随后多个要素均需要规定的要求。
- 涉及总体原则、总则、原则或总体要求时，宜设置相应要素。
- 总体原则/总则/原则应使用陈述或推荐型条款，不应包含要求型条款。
- 总体要求应使用要求型条款。

### 4.11 核心技术要素

- 核心技术要素是各种功能类型标准的标志性要素。
- 标准功能类型不同，核心技术要素和条款类型也不同。
- 核心技术要素的具体编写应遵守 GB/T 20001（所有部分）的规定。
- 规范标准的“要求”和“证实方法”应闭环：要求可被证实，证实方法能判定要求是否满足。
- 规程标准的程序要求应明确可追溯或证实的方法。
- 指南标准应以推荐和陈述为主，不宜伪装成强制要求。

### 4.12 其他技术要素

- 可根据情况设置试验条件、仪器设备、取样、标志、标签和包装、标准化项目标记、计算方法等。
- 涉及标准化项目标记时，应符合附录 B 规定。

### 4.13 参考文献

- 参考文献列出资料性引用的文件清单，以及其他供参阅的信息资源。
- 有资料性引用文件时，应设置参考文献。
- 参考文献应置于最后一个附录之后。
- 参考文献不应分条；清单可通过描述性标题分组，标题不编号。
- 每个参考文件或信息资源前应在方括号中给出序号。
- md2std Markdown 输入中不手写 `[1]`、`[2]` 等参考文献序号；只写参考文献正文，最终 Word 版式中的方括号序号由生成器/样式自动处理。
- 清单内容、排列顺序、在线文献列出方式应符合 8.6.3 相关规则；国际文件、国外文件不必给出中文译名。
- GB/T 1.1 通常作为前言中的起草依据说明；除非正文确有资料性引用或作为供参阅信息资源的必要，不宜机械列入参考文献。

### 4.14 索引

- 索引用于通过关键词检索文件内容。
- 为方便文件使用者需要设置索引时，索引应作为最后一个要素。
- 索引项以文件中的关键词作为索引标目，并给出规范性要素中对应的章、条、附录、图、表编号。
- 索引项通常按关键词的汉语拼音字母顺序编排。
- 为便于检索，可在同一汉语拼音首字母的索引项之上标出相应字母；字母应与关键词首字母一致，例如 `允许开采量` 应归入 `Y`，`循环深度` 应归入 `X`。
- 电子文本的索引宜自动生成。

## 5. 条款类型与要素表述

### 5.1 条款类型

条款类型包括要求、指示、推荐、允许和陈述。表述应使使用者能清楚识别符合性所需满足的要求或执行的指示，并能区分推荐、允许和陈述。

| 条款类型 | 典型表述 | 用途 |
| --- | --- | --- |
| 要求 | `应`、`不应`、条件性 `应` | 规定必须满足或禁止的内容 |
| 指示 | 祈使句，或按附录 C 可用表述 | 规定需要执行的动作或程序 |
| 推荐 | `宜`、`不宜` | 建议采用或避免 |
| 允许 | `可`、`不必` | 表示许可或无需 |
| 陈述 | 陈述句 | 提供事实、说明或信息 |

### 5.2 附加信息

- 附加信息包括示例、注、脚注、图表脚注、规范性引用文件和参考文献清单、目次、索引等。
- 除图表脚注外，附加信息宜表述为事实陈述，不应包含要求、指示、推荐或允许型条款。
- 条文注、术语条目注、图中的注和表中的注均属于注，不应写入 `应`、`不应`、`宜`、`不宜`、`可`、`不必` 等条款表述；需要规定要求时，应移入条款、表中段或规范性附录。
- 图表脚注可包含要求型条款，但应通过适当能愿动词或句子语气明确条款类型。
- 示例中为说明条款表述而包含要求、指示、推荐或允许时，不视为违规；必要时可用线框区分。

### 5.3 通用内容

- 某章/条通用内容宜作为该章/条中最前面的一条。
- 可根据内容使用 `通用要求`、`通则`、`概述` 作为标题。
- 通用要求用于规定涉及多条的要求，均应使用要求型条款。
- 通则用于规定共性内容，至少包含要求型条款，也可包含其他类型条款。
- 概述用于给出陈述或说明，应使用陈述型条款，不应包含要求、指示或推荐；除非确有必要，通常不设置概述。

### 5.4 常用词

- `遵守` 用于人员或组织采取行动的条款。
- `符合` 用于产品/系统、过程或服务特性达到文件或要求的条款。
- `尽可能`、`尽量`、`考虑`、`优先考虑`、`充分考虑`、`避免`、`慎重` 等不宜与 `应` 连用表示要求；建议与 `宜` 连用表示推荐。
- `通常`、`一般`、`原则上` 不宜与 `应`、`不应` 连用；可与 `宜`、`不宜` 连用。
- 可用“……情况下应……”“只有/仅在……时，才应……”“根据……情况，应……”“除非……特殊情况，不应……”表示有前提条件的要求；前提条件应清楚明确。

### 5.5 全称、简称和缩略语

- 文件中应仅使用组织机构正在使用的全称和简称或原文缩写。
- 需要使用简称时，首次出现应在全称后圆括号中给出简称，之后使用简称。
- 未设置缩略语清单但需要使用拉丁字母缩略语时，首次出现应给出中文词语或解释，并将缩略语置于圆括号中。
- 拉丁字母缩略语宜慎用，只有不引起混淆时才使用。
- 缩略语宜由大写拉丁字母组成，每个字母后不加下脚点；历史或技术原因形成的约定俗成写法可例外。

### 5.6 数、数值、尺寸和公差

- 表示物理量的数值应使用阿拉伯数字并跟随法定计量单位符号。
- 数字用法应遵守 GB/T 15835。
- 乘积、向量积和笛卡尔积宜使用符号 `×`；标量乘积、组合单位等可使用居中圆点 `·`。
- 图坐标轴和表头可使用 `v/(km/h)`、`l/m` 等量与单位比值形式。
- 尺寸和公差应无歧义。
- 长宽高等尺寸应写为每个数值都带明确定义的单位或清楚的组合形式，避免只在最后写一次单位导致歧义。
- 公差通常用最大值、最小值、带公差值或范围值表示。
- 温度等范围值应避免写成只在末端出现单位的歧义形式，例如应表达为 `0 ℃～10 ℃` 这类完整范围。
- 百分率公差应明确区分范围值和带公差值，如范围用 `63 %～67 %`，带公差值用 `(65±2) %`。
- 平面角宜用度表示；度、分、秒的单位符号紧跟数值。

### 5.7 量、单位和符号

- 文件中的量、单位及符号应从 GB/T 3101、GB/T 3102、ISO 80000、IEC 80000、GB/T 14559、IEC 60027 等规定中选择。
- 进一步单位使用规则见 GB 3100。
- 变量符号应使用斜体，其他符号使用正体。
- 除平面角的度、分、秒外，其他单位符号前应空四分之一个汉字的间隙。

## 6. 引用和提示

### 6.1 基本原则

- 现行有效文件或文件自身其他条款已经包含且适用的内容，应通过引用或提示使用，不宜抄录。
- 引用和提示有助于避免重复、不协调、篇幅过大和抄录错误。
- 在线引用文件应提供足以识别和定位来源的信息，宜提供第一手来源、访问方法和完整网址。
- 修订文件时应确认所有引用文件的有效性。

### 6.2 文件自身称谓

- 文件中称呼文件自身时使用 `本文件`。
- 分为部分的文件中，某部分需要称呼所有部分时，使用文件编号整体称谓，如 `GB/T XXXXX`。

### 6.3 提及文件具体内容

- 不应提及页码，应提及内容编号。
- 章或条可写作 `第4章`、`5.2`、`9.3.3 b)`、`A.1` 等。
- 附录写作 `附录C`。
- 图或表写作 `图1`、`表2`。
- 数学公式写作 `公式（3）` 或 `10.1中的公式（5）`。

### 6.4 注日期引用

- 注日期引用表示被引用文件的指定版本适用。
- 不能确定是否可接受被引用文件未来所有变化，或提及被引用文件具体章、条、图、表、附录编号时，应注日期。
- 注日期引用应指明年份；同一年发布多个版本时应指明年份和月份。
- 引用具体内容时应提及内容编号。
- 被引用文件后续发布修改单或修订版且经评估有必要更新时，可通过发布引用文件自身修改单更新引用。

### 6.5 不注日期引用

- 不注日期引用表示被引用文件最新版本（包括修改单）适用。
- 只有能够接受所引用内容未来所有变化，且引用完整文件，或未提及具体内容编号时，才可不注日期。
- 不注日期引用不应指明年份，只提及文件代号和顺序号。
- 引用一个文件所有部分时，应在文件顺序号后标明 `（所有部分）`。
- 不注日期引用但实际需要引用具体内容而未提及编号时，可在脚注中说明涉及的现行文件章、条、图、表或附录编号。

### 6.6 规范性引用和资料性引用

- 规范性引用的文件内容构成引用文件中必不可少的条款。
- 由要求型或指示型条款提及文件，通常属于规范性引用。
- 规范标准中由 `按` 或 `按照` 提及试验方法类文件，属于规范性引用。
- 指南标准中由推荐型条款提及文件，属于规范性引用。
- `术语和定义` 引导语中提及文件，属于规范性引用。
- 所有规范性引用文件应列入 `规范性引用文件`。
- 资料性引用用于帮助理解或使用文件，不构成必不可少条款。
- 凡不属于规范性引用的提及通常属于资料性引用。
- 资料性引用文件应列入 `参考文献`。
- 法律法规可在必要时资料性提及，也可用 `必须` 陈述外部法律约束；这种外部约束不是文件自身规定的条款，通常宜与标准条款分条表述。

### 6.7 标明来源

- 特殊情况下确需抄录其他文件少量内容时，应在抄录内容下方或之后准确标明来源。
- 来源格式应写明文件编号和章/条编号或条目编号。
- 术语条目改写来源定义时，应在来源处说明有修改。

### 6.8 被引用文件限制

- 被规范性引用的文件原则上应是国家、行业或国际标准化文件。
- 其他正式发布的标准化文件或文献，只有经归口标准化技术委员会或审查会议确认满足权威性、可获得性、修订通知、商业条款和专利许可等条件时，方可规范性引用。
- 起草文件时不应引用不能公开获得的文件。
- 起草文件时不应引用已被代替或废止的文件。
- 起草文件时不应规范性引用法律、行政法规、规章和其他政策性文件。
- 不应写“应符合国家有关法律法规”这类普遍性要求。

### 6.9 提示文件自身内容

- 需要使用者遵守、履行或符合文件自身具体条款时，应使用适当能愿动词或句子语气并提及内容编号；这属于规范性提示。
- 需要使用者参看文件自身具体内容时，应使用 `见` 加内容编号，不应使用“见上文”“见下文”等形式；这属于资料性提示。

## 7. 附录

- 附录用于承接和安置不便在正文、前言或引言中表述的内容，是对正文、前言或引言的补充或附加。
- 规范性附录给出正文的补充或附加条款。
- 资料性附录给出有助于理解或使用文件的附加信息。
- 附录的规范性或资料性作用应在目次和附录编号之下标明。
- 正文、前言或引言中移出内容形成附录时，应在移出位置用适当表述指明附录编号。
- 要求型或指示型条款指明的附录属于规范性附录。
- 规范标准中由 `按` 或 `按照` 指明试验方法的附录属于规范性附录。
- 指南标准中由推荐型条款指明的附录属于规范性附录。
- 其他表述形式指明的附录通常属于资料性附录。
- 附录应位于正文之后、参考文献之前。
- 附录顺序取决于其被移作附录之前所处位置的前后顺序。
- 每个附录均应有编号。附录编号由 `附录` 和从 A 开始的大写拉丁字母组成；只有一个附录时仍为 `附录A`。
- 附录编号之下应标明 `(规范性)` 或 `(资料性)`，再下方为附录标题。
- 附录可分条并继续细分。
- 每个附录中的条、图、表、公式编号均从 1 重新开始，并加附录字母前缀，如 `A.1`、`图A.1`、`表A.1`、`公式（A.1）`。
- 附录中不准许设置 `范围`、`规范性引用文件`、`术语和定义` 等内容。

## 8. 图

- 当图形比文字更便于理解相关内容时，宜使用图。
- 在将内容图形化之处，应通过适当能愿动词或句子语气指明图所表示的条款类型，并提及图编号。
- 每幅图均应编号。图编号由 `图` 和从 1 开始的阿拉伯数字组成；只有一幅图仍编号 `图1`。
- 图编号从引言开始连续到附录之前，与章、条、表编号无关。
- 每幅图宜有图题；文件中图有无图题应一致。
- 需要转页接排时，续图页应重复图编号，后接图题（可选）和 `(续)` 或 `(第#页/共*页)`。
- 续图应重复关于单位的陈述。
- 图中表示角度量或线性量的字母符号应符合量和单位标准，必要时用下标区分。
- 表示各种长度时宜使用 `l1`、`l2`、`l3` 等符号系列，不宜使用 A、B、C 或 ab、c 等。
- 图中所有量单位相同时，应在图右上方给出如 `单位为毫米` 的单位陈述。
- 图中应优先使用标引序号或图脚注代替文字描述；文字描述在标引序号说明或图脚注中给出。
- 曲线图坐标轴上的标记不应以标引序号代替。
- 流程图和组织系统图中允许使用文字描述。
- 分图会增加编排和管理复杂性，宜避免使用；只有图的表示或理解特别需要时才使用。
- 分图只准许一个层次，使用后带半圆括号的小写拉丁字母编号，如 `a)`、`b)`。
- 不应使用 `1.1`、`1-1` 等分图编号。
- 如果每个分图都有各自标引序号说明、注或脚注，应调整为单独的图。

## 9. 表

- 当表格比文字更便于理解内容时，宜使用表。
- 表越简单越好；与其把过多内容合并成复杂表，不如拆成多个表。
- 在将内容表格化之处，应通过适当能愿动词或句子语气指明表所表示的条款类型，并提及表编号。
- 不准许将表再细分为分表，如 `表2a`、`表2b`。
- 不准许表中套表或表中含带表头的子表。
- 每个表均应编号。表编号由 `表` 和从 1 开始的阿拉伯数字组成；只有一个表仍编号 `表1`。
- 表编号从引言开始连续到附录之前，与章、条、图编号无关。
- 每个表宜有表题；文件中表有无表题应一致。
- 表需要转页接排时，续表页应重复表编号，后接表题（可选）和 `(续)` 或 `(第#页/共*页)`。
- 续表应重复表头和关于单位的陈述。
- 每个表应有表头。表头通常位于表上方，特殊情况下可位于左侧边栏。
- 表中各栏/行单位不完全相同时，宜将单位符号置于相应表头中量的名称之下。
- 表中所有量单位相同时，应在表右上方用 `单位为……` 代替各栏单位符号。
- 表头中不准许使用斜线。
- 表中的注置于表内下方、表脚注之上。
- 表中内容为数字时，数字宜居中；同列数字宜个位对齐或小数点对齐；范围连接号宜上下对齐。
- 表中相邻数字或文字相同时，不应使用 `同上`、`同左`，可用通栏表示或写具体内容。
- 表的单元格中不应有空格；无内容时用一字线形式的连接号表示。

## 10. 数学公式

- 需要用符号表示量之间关系时宜使用数学公式。
- 需要引用或提示公式时，应使用带圆括号、从 1 开始的阿拉伯数字编号。
- 公式编号从引言开始连续到附录之前，与章、条、图、表编号无关。
- 不准许将数学公式进一步细分，如 `(2a)`、`(2b)`。
- 数学公式通常使用量关系式，变量应由字母符号表示。
- 除非已经在 `符号和缩略语` 中列出，否则应在公式后用 `式中：` 引出字母符号含义解释。
- 特殊情况下使用数值关系式时，应解释表示数值的符号并给出单位。
- 同一文件中，同一个符号不应既表示物理量，又表示该物理量对应的数值。
- 数学公式不应使用量的名称或描述量的术语表示。
- 多字母缩略术语不应代替量符号。
- 数学公式中不应使用单位符号。
- 同一文件中同一个符号不宜代表不同的量；可用下标区分相关概念。
- 条文中宜避免使用多于一行的表示形式。
- 数学公式中宜避免多于一个层次的上标或下标，并避免多于两行的表示形式。

## 11. 示例、注和脚注

### 11.1 示例

- 示例属于附加信息，用具体例子帮助理解或使用文件。
- 示例宜置于所涉及的章或条之下。
- 一个示例时标明 `示例：`。
- 多个示例时宜从 `示例1：` 连续编号。
- 示例不宜单独设章或条。
- 示例较多或篇幅较大，尤其包含多个图或表时，宜以 `示例` 为标题形成资料性附录。
- 不宜每个示例、图或表各自编为单独附录。
- 示例与条款易混淆或涉及编排格式时，可置于线框内。

### 11.2 注

- 注属于附加信息，只给出有助于理解或使用文件内容的说明。
- 条文中的注宜置于所涉及的章、条或段之下。
- 术语条目中的注应置于示例之后。
- 图中的注应置于图题和图脚注之上。
- 表中的注应置于表内下方、表脚注之上。
- 一个注时标明 `注：`。
- 多个注时从 `注1：` 连续编号；同一章、条、术语条目、图或表内分别编号。

### 11.3 脚注

- 条文脚注只给出针对条文中特定内容的附加说明，宜尽可能少用。
- 条文脚注置于相关页面左下方细实线之下。
- 条文脚注编号从前言开始全文连续，编号形式为后带半圆括号的阿拉伯数字，如 `1)`、`2)`。
- 特殊情况下，为避免与上标数字混淆，可用一个或多个星号代替脚注数字编号。
- 图脚注置于图题之上，紧跟图中的注。
- 表脚注置于表内最下方，紧跟表中的注。
- 图表脚注使用从 `a` 开始的上标小写拉丁字母；每个图或表单独编号。
- 图表脚注除附加信息外，也可包含要求型条款，因此应使用适当能愿动词或句子语气明确条款类型。

## 12. 商品名、商标、专利和重要提示

### 12.1 商品名和商标

- 文件中应给出产品的正确名称或描述，不应给出商品名或商标。
- 特定产品专用商品名或商标，即使通常使用，也宜尽可能避免。
- 特殊情况下无法避免使用商品名或商标时，应指明其性质，如注册商标或商标。
- 如果适用文件的产品目前只有一种，可给出商品名或商标，但应以脚注说明该信息仅为方便使用者，不表示认可该产品。

### 12.2 专利

- 起草过程中应按相关程序处理涉及专利的信息征集和说明。
- 尚未识别出涉及专利时，前言中应给出相应提示。
- 已识别出涉及专利时，应在引言中说明专利信息、许可声明或联系方式；内容较多时可移作附录。
- 征求意见稿和送审稿封面显著位置应给出征集专利信息的提示。

### 12.3 重要提示

- 涉及整个文件内容的重要事项，可在正文首页文件名称和范围之间设置重要提示。
- 涉及人身安全或健康时，应根据风险程度使用 `危险：`、`警告：`、`注意：` 或 `重要提示：`。
- 重要提示不替代正文中的具体安全要求或说明。

## 13. 编排格式和 Word 版式审校依据

本章记录 GB/T 1.1 的编排规则，用于审校和验收生成结果；不规定 md2std 的模板实现方式。

### 13.1 页面和页码

- 出版文件采用 A4 开本，幅面尺寸 210 mm × 297 mm；特殊情况下如图表不能缩小时，可加宽和/或延长。
- 不同页面格式和字号字体应符合相应附录规定。
- 单数页和双数页格式分别设置。
- 从目次开始，书眉应编排文件编号。
- 从目次页到正文首页前，用从 `I` 开始的正体大写罗马数字编页码。
- 正文首页起，用从 `1` 开始的阿拉伯数字另编页码。
- 正文首页应从单数页开始。
- 正文首页中文件名称由多个元素组成时，各元素之间空一个汉字间隙；名称较长时可多行编排。
- 重要提示应空两个汉字起排，回行顶格。
- 末页中文件最后一个要素内容之下应有终结线；终结线为居中粗实线，长度为版心宽度四分之一；终结线应与最后一个要素内容位于同一页。

### 13.2 章、条、段和列项编排

- 章、条编号顶格起排，空一个汉字间隙后接排标题。
- 章编号和章标题单独占一行，上下各空一行。
- 条编号和条标题单独占一行，上下各空半行。
- 无标题条编号之后空一个汉字间隙接排条文。
- 段文字空两个汉字起排，回行顶格。
- 第一层次列项的破折号或字母编号空两个汉字起排，文字及回行置于版心左边第五个汉字位置。
- 第二层次列项的间隔号或数字编号空四个汉字起排，文字及回行置于版心左边第七个汉字位置。

### 13.3 要素编排

- 封面格式按文件类型采用相应格式。
- 文件名称由多个元素组成时，各元素之间空一个汉字间隙；多行编排行间距应符合格式要求。
- 文件名称英文译名各元素首字母大写，其余小写；元素之间使用一字线形式连接号。
- 文件编号中，文件代号与顺序号之间空半个汉字间隙；顺序号与年份号之间用一字线连接号 `—`，不使用半字线 `-` 或 en dash `–`。
- 被代替文件编号置于文件编号之下另行编排，右端对齐，前加 `代替`。
- 封面 ICS 号和 CCS 号分上下两行编排，左端对齐；行业标准备案号置于 CCS 号之下。
- 前言和引言均应另起一面。
- 规范性引用文件中所列文件均空两个汉字起排，回行顶格，文件之后不加标点符号；文件编号或 `（所有部分）` 与文件名称之间空一个汉字间隙。
- 术语条目编号顶格起排，单独占一行，上下无空行。
- 英文对应词位于术语之后，与术语之间空一个汉字间隙；除原文需要大写外，英文对应词均小写。
- 除条目编号、英文对应词外，术语条目各项内容另行空两个汉字起排，定义回行顶格。
- 参考文献和索引均应另起一面；标题居中。
- 参考文献条目空两个汉字起排，回行顶格，文件之后不加标点符号。
- 参考文献序号与文件编号之间、文件编号或 `（所有部分）` 与文件名称之间均空一个汉字间隙。

### 13.4 附录编排

- 每个附录均应另起一面。
- 附录编号、附录作用 `(规范性)` 或 `(资料性)`、附录标题各占一行，居中编排。
- 附录中的条、图、表和公式使用附录字母前缀重新编号。

### 13.5 图、表和公式编排

- 每幅图与其前面的条文、每个表与其后面的条文之间均宜空一行。
- 图编号和表编号之后均空一个汉字间隙接排图题或表题。
- 图编号和图题置于图下居中；表编号和表题置于表上居中。
- 图编号和图题、表编号和表题的上下各空半行。
- 表外框线、表头框线以及表中注、表脚注所在框线均为粗实线。
- 表中段落宜空一个汉字起排，回行顶格，段后不必加标点符号。
- 数学公式另行居中编排。
- 较长公式应在等号、加号、减号、正负号之后，必要时在乘号、居中圆点或斜线之后回行。
- 公式中的分数线主线与辅线应明确区分，主线应与等号取平。
- 公式编号右端对齐，公式与编号之间用连接符号连接。
- `式中：` 空两个汉字起排，单独占一行。
- 公式符号说明按先左后右、先上后下顺序分行说明，每行空两个汉字起排，用破折号与释文连接，回行与上一行释文对齐，各行破折号对齐。

### 13.6 注、脚注、示例和单位符号编排

- 条文注、术语条目注、图中注、表中注均另行空两个汉字起排，回行与注内容文字位置左对齐。
- 条文脚注另行空两个汉字起排，文字及回行置于版心左边第五个汉字位置；分隔线长度为版心宽度四分之一。
- 图脚注另行空两个汉字起排，文字及回行置于版心左边第四个汉字位置。
- 表脚注另行空两个汉字起排，文字及回行置于表左框线第四个汉字位置。
- 示例另行空两个汉字起排；`示例：` 或 `示例×：` 宜单独占一行；文字类示例回行宜顶格。
- 区分示例的线框应为细实线。
- 变量符号用斜体，其他符号用正体。
- 平面角的度、分、秒单位符号紧跟数值；其他单位符号前空四分之一个汉字间隙。
